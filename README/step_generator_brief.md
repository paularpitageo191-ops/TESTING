It acts as the **execution bridge**, mapping:

## 🧠 Core Functionalities

### 1. Feature Parsing
- Reads Gherkin feature file
- Supports:
  - Background
  - Scenario
  - Scenario Outline
- Expands Scenario Outline into unique scenarios

KEY FIX:
Ensures unique scenario names → prevents Playwright crash

---

### 2. Step Understanding (LLM)
Each step is analyzed using LLM:

Output:
- action_type
- intent
- value
- confidence

---

### 3. Semantic Mapping (Qdrant)
- Matches step intent to UI elements
- Uses embeddings + similarity search

---

### 4. URL Resolution (Critical)
Navigation steps resolve URL via:

1. Qdrant UI memory  
2. Step text (if URL present)  
3. BASE_URL fallback  

---

### 5. Smart Action Mapping

All steps convert to:

basePage.smartAction(intent, value)

Handles:
- click
- fill
- verify
- navigation

---

### 6. TypeScript Code Generation

Generates:
- Playwright test.describe()
- test() per scenario
- Background setup
- Step execution
