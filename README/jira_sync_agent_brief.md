# Jira Sync Agent — Technical Brief

## 📁 File
jira_sync_agent.py

## 🎯 Purpose
The Jira Sync Agent is the entry point (Phase 0) of the TEA pipeline.
It pulls structured and unstructured requirement data from Jira and converts it into a normalized requirements.json.

## 🔄 Position in Pipeline
Phase 0 → Jira Sync Agent  
Phase 1 → Vectorization + PRD  
Phase 2 → DOM + Gherkin  
Phase 3 → Test Execution  
Phase 4 → Reporting  

## 📥 Inputs

### CLI
python3 jira_sync_agent.py --issue SCRUM-86 --project SCRUM

### Environment Variables (.env)
JIRA_BASE_URL  
JIRA_EMAIL  
JIRA_API_TOKEN  
JIRA_PROJECT_KEY  

### External Inputs
- Jira issues (Story/Epic)
- Attachments (CSV, Excel, PDF, Text)

## 📤 Outputs

### Primary
docs/requirements_<ISSUE>.json

### Secondary
docs/inbox/requirements_<ISSUE>.json

## 🧠 Core Functionalities

1. Fetch Jira issue using REST API  
2. Identify Epic/Story relationships  
3. Download attachments  
4. Parse attachments:
   - CSV → csv
   - Excel → pandas
   - PDF → PyPDF2
   - Text/MD/JSON → raw
5. Normalize into requirement objects  
6. Build consolidated text

## ⚠️ Failure Points

- Jira authentication failure  
- Missing pandas/PyPDF2  
- Unsupported file formats  
- Large attachments
