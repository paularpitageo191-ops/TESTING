# Self-Healing Agent — Technical Brief

## 📁 File
self_healing_agent.py

## 🎯 Purpose
Orchestrates post-execution intelligence:
- Parses Playwright results
- Detects UI drift via Qdrant similarity
- Classifies outcomes (PASS | FAIL | DRIFT | FLAKY)
- Pushes results to Zephyr Scale
- Creates Jira bugs with evidence
- Tracks flakiness over time

## 🧠 Core Functionalities

### 1. Playwright Result Parsing
- Loads latest JSON result
- Flattens suites → test cases
- Extracts:
  - status
  - duration
  - error
  - screenshot

---

### 2. Flakiness Tracking
- Maintains historical DB
- Detects intermittent failures:
  failure rate between 15%–85%

---

### 3. UI Drift Detection (Key Feature)

Each failed step → embedding → Qdrant search

Score → Drift Type:

| Score | Drift |
|------|------|
| ≥0.55 | None |
| 0.40–0.54 | Cosmetic |
| 0.25–0.39 | Structural |
| <0.25 | Functional |

---

### 4. Classification Engine

Each test becomes:

- PASS
- FAIL
- DRIFT (UI change detected)
- FLAKY (intermittent)

---

### 5. Zephyr Integration

Creates:
- Test Cycle per run
- Test Execution per test

Adds:
- Comments (error + drift info)
- Screenshots
- Links execution → Jira Story

---

### 6. Jira Bug Creation

For FAIL / DRIFT:
- Creates Bug
- Links Bug → Story
- Adds:
  - Error logs
  - Drift analysis table
  - Screenshot

---

### 7. Run Summary

Generates:
docs/healing-logs/<PROJECT>-run-*.json

Contains:
- totals
- per-test status
- drift info
