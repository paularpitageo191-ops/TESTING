# BasePage — Technical Brief

## 📁 File
BasePage.ts

## 🎯 Purpose
Core execution engine for Playwright tests.

Provides a **single intelligent API (`smartAction`)** that:
- Executes all UI actions
- Uses Qdrant semantic search instead of hardcoded selectors
- Enables self-healing automation

## 🧠 Core Functionalities

### 1. smartAction (Main API)

Single entry point for:
- click
- fill
- verify
- navigate

Automatically determines action type from intent

---

### 2. Semantic Element Search

Uses:
- Embedding (Ollama)
- Qdrant search

Matches:
intent → UI element

No hardcoded selectors required

---

### 3. Self-Healing Locator System

Builds multiple locator strategies:
- XPath
- CSS
- ID / name
- Label / placeholder
- Role-based
- Text-based

Tries all until match found

---

### 4. URL Resolution (Critical)

For navigation:
- Extract URL from intent
- OR resolve via Qdrant
- OR fallback to BASE_URL

---

### 5. Intelligent Verify Handling

Handles:
- Page navigation validation
- Text validation
- Error message detection

Prevents false positives:
- Error messages ≠ URL assertions

---

### 6. Drift Handling

If element not found:
- Falls back to semantic search
- Logs healing attempt
- Provides debug output

---

### 7. Healing Tracking

Tracks:
- intent
- selector used
- confidence

Accessible via:
getHealingStats()
getHealingLog()

---

## 📁 File Structure Impact

## ⚠️ Failure Points

- Qdrant empty → no element resolution
- Embedding failure → search fails
- Incorrect intent → wrong action
- Missing BASE_URL → navigation fallback fails
