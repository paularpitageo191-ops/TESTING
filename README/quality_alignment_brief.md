# Quality Alignment Agent — Technical Brief

## 📁 File
quality_alignment.py

## 🎯 Purpose
This agent validates alignment between:
- Requirements (Jira + Qdrant)
- Live UI (DOM capture)

It generates:
- Gherkin feature file (LLM-driven)
- Quality alignment report (drift + confidence + coverage)

## 🧠 Core Functionalities

### 1. Requirements Loading
- Fetches from Qdrant
- Falls back to inbox JSON
- Deduplicates text

---

### 2. DOM Loading
- Loads latest DOM snapshot
- Builds summarized UI representation

---

### 3. DOM Vector Upload (Critical)
- Uploads UI elements to Qdrant
- Stores URL per element

WHY:
Enables correct navigation in test steps

---

### 4. Gherkin Generation (LLM)
- Uses LLM to generate feature file
- Ensures:
  - Unique scenario titles
  - Positive + negative coverage
  - Realistic data
  - Mapping to automation steps

Fallback:
- Generates basic scenarios if LLM fails

---

### 5. Semantic Cross-Reference
- Matches requirements ↔ DOM using embeddings
- Uses Qdrant search

---

### 6. Drift Analysis
Checks if requirement concepts exist in UI

Output:
- PRESENT
- MISSING (drift)

---

### 7. Confidence Validation
Maps UI elements → requirements

Scores:
- HIGH (>= 0.6)
- MEDIUM (>= 0.3)
- LOW (< 0.3)

LOW → flagged for self-healing

---

### 8. Selector Coverage
Validates selectors from:
docs/selectors.csv

---

### 9. Report Generation
Creates structured report with:
- Cross-reference results
- Drift analysis
- Validation scores
- Self-healing suggestions
