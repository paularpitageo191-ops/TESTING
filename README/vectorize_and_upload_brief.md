# Vectorization & Upload Agent — Technical Brief

## 📁 File
vectorize_and_upload.py

## 🎯 Purpose
This agent converts requirements and UI data into vector embeddings and stores them in Qdrant.

It acts as the **semantic memory layer** of the system, enabling:
- Intelligent step generation
- Self-healing
- Context-aware test execution

## 🧠 Core Functionalities

### 1. Requirement Processing
- Extracts text from:
  - JSON (Jira output)
  - CSV / Excel
  - PDF
- Converts ADF → plain text
- Splits into chunks

---

### 2. Business Intent Detection
Each text chunk is classified:
- requirement
- capability
- test_case

---

### 3. Embedding Generation
Uses:
llm_gateway → provider-agnostic

Outputs:
vector (dimension depends on model)

---

### 4. Qdrant Upload
- Creates collection per project
- Stores:
  - text
  - embedding
  - metadata
  - URL references

---

### 5. DOM Upload (Critical Feature)
- Uploads UI elements into vector DB
- Stores **page URL in every point**

WHY IMPORTANT:
Without URL → navigation fails in step generation

---

### 6. URL Extraction
Extracts URLs from requirements text:
Used later for:
- Navigation resolution
- Step generation

---

### 7. PRD Generation
Creates:
docs/requirements/<PROJECT>_PRD.md

Includes:
- Requirement summary
- Vector collections
- Healing strategy

## ⚠️ Failure Points

- Embedding failure → empty vectors
- Qdrant dimension mismatch
- Missing URL in DOM → navigation issues
- Large files → performance issues
- Missing dependencies (pandas, PyPDF2)
